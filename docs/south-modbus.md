# Modbus TCP 南向采集插件说明文档

`Modbus TCP` 是一种应用于工业自动化领域的标准通信协议。本边缘网关集成的 Modbus TCP 南向采集插件，旨在高效率地从支持 Modbus TCP 协议的 PLC、温控器、电表及各种传感器等硬件设备中轮询采集点位数据。

---

## 1. 南向采集通道配置

在“南向采集”页面新建 Modbus TCP 通道时，需要配置底层网络连接参数：

| 参数名称 | 英文 Key | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **主机地址** | `host` | String | 是 | 目标 Modbus 设备的 IP 地址，例如 `192.168.1.100` |
| **端口** | `port` | Number | 是 | 端口号，Modbus TCP 默认通常为 `502` |
| **从站号** | `unitId` | Number | 是 | 单元 ID（站号），默认为 `1`。用于在 TCP 链路中标识具体的物理从站 |

---

## 2. 点位（Tag）映射配置

在具体的采集组中添加点位时，Modbus TCP 插件支持丰富的寄存器类型和数据处理机制：

### 2.1 寄存器地址格式 (`address`)

点位的 `address` 用于指示要读取的 Modbus 寄存器类型及偏移地址，格式如下：

*   **`0xxxx` (线圈寄存器 / Coils)**：可读可写，对应单路开关控制。例如 `00001` 表示第 1 个线圈。
*   **`1xxxx` (离散输入寄存器 / Discrete Inputs)**：只读，对应数字量输入。例如 `10005`。
*   **`3xxxx` (输入寄存器 / Input Registers)**：只读，对应模拟量输入。例如 `30010`。
*   **`4xxxx` (保持寄存器 / Holding Registers)**：可读可写，最常用的配置和测量数据区。例如 `40001`。

> [!NOTE]
> 程序会自动解析前缀（`0`/`1`/`3`/`4`）并提取后续十进制数作为偏移量地址。地址采用从 1 开始的工业标准习惯。

### 2.2 数据类型与字节数 (`type`)

插件支持以下标准数据类型，并自动分配所需的寄存器长度（每个寄存器占 2 字节/16位）：

| 数据类型 | 占用寄存器数 | 说明 |
| :--- | :---: | :--- |
| **int16** / **uint16** | 1 (16 bits) | 16 位有符号 / 无符号整数 |
| **int32** / **uint32** | 2 (32 bits) | 32 位有符号 / 无符号整数 |
| **int64** / **uint64** | 4 (64 bits) | 64 位有符号 / 无符号整数 |
| **float32** | 2 (32 bits) | 单精度浮点数 |
| **float64** | 4 (64 bits) | 双精度浮点数 |
| **bit** | 0.5 (1 bit) | 保持寄存器内的单 bit 位（支持 `4xxxx.y` 语法，用于提取特定位） |

### 2.3 字节序配置 (`byteOrder`)

对于多于 1 个寄存器（32位及以上）的数据，由于不同硬件厂商的端序设计不同，需要配置字节序进行编解码转换：

*   **`AB` / `BA`**：16 位数据的端序。
*   **`ABCD`**：大端序（Big-Endian），最标准的端序。
*   **`BADC`**：16位字内字节交换（字节反转）。
*   **`CDAB`**：字交换（Middle-Endian，常用于某些 PLC，如西门子/施耐德）。
*   **`DCBA`**：小端序（Little-Endian）。

### 2.4 缩放系数 (`decimal`)

从寄存器读出的原始数值，插件会根据配置的 `decimal`（缩放因子）自动进行转换：
\[ \text{实际值} = \text{原始值} \times \text{decimal} \]
*例如：电表电压寄存器读出 `2203`，配置 `decimal = 0.1` 后，最终上报的实际物理值将自动计算为 `220.3`。*

## 3. 高级性能优化：连续寄存器打包读取

为了解决高频轮询大批量点位时网络拥堵和硬件响应不及时的问题，本插件内置了**智能合并读取优化算法**：

*   **区间判定**：调度器在执行采集时，会扫描同组下的所有 Modbus 点位，对同类寄存器区（如 Holding Registers）按地址进行排序。
*   **智能合并**：若点位地址连续或间隔很小，插件会将其自动合并为一个大型的 Modbus 读取事务（单个批量请求），一次性将大块字节数据读回，再在内存中根据各个点位的偏移地址和字节序进行切片解析。
*   **效果明显**：相比于逐点单请求轮询，连续寄存器打包读取可以降低 90% 以上的 TCP 包交互频率，保障边缘端的极低 CPU 消耗和高连通稳定性。
