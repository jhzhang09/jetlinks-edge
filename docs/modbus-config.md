# Modbus 配置说明

## 地址格式

采用 PLC 风格 5 位数字地址：

| 前缀 | 区域 | 读写 | 功能码 |
|---|---|---|---|
| 0xxxxx | 线圈 (Coil) | 读写 | 读 FC01 / 写单 FC05 / 写多 FC15 |
| 1xxxxx | 离散输入 (Discrete Input) | 只读 | FC02 |
| 3xxxxx | 输入寄存器 (Input Register) | 只读 | FC04 |
| 4xxxxx | 保持寄存器 (Holding Register) | 读写 | 读 FC03 / 写单 FC06 / 写多 FC16 |

示例：

- `00001` 第 0 个线圈
- `10001` 第 0 个离散输入
- `30001` 第 0 个输入寄存器
- `40001` 第 0 个保持寄存器
- `40100` 第 99 个保持寄存器

## 数据类型

| 类型 | 占用字节 | 字节序 | 说明 |
|---|---|---|---|
| bool | 1 | - | 寄存器任一字节非 0 即 true |
| int16 | 2 | AB / BA | 16 位有符号整数 |
| uint16 | 2 | AB / BA | 16 位无符号整数 |
| int32 | 4 | ABCD / BADC / CDAB / DCBA | 32 位有符号整数 |
| uint32 | 4 | 同上 | 32 位无符号整数 |
| int64 | 8 | ABCDEFGH... | 64 位有符号整数 |
| uint64 | 8 | 同上 | 64 位无符号整数 |
| float32 | 4 | ABCD / BADC / CDAB / DCBA | IEEE 754 单精度 |
| float64 | 8 | ABCDEFGH... | IEEE 754 双精度 |
| string | 2*N | - | 按 N 个寄存器读取，直接按字节解码 |
| bytes | 2*N | - | 按 N 个寄存器读取，返回原始字节 |

## 字节序

- AB / BA：2 字节类型。AB = 大端（高字节在前），BA = 小端。
- ABCD：4 字节类型的大端。
- BADC：4 字节类型的大端字节交换（AB CD → BA DC），常见于三菱/西门子。
- CDAB：4 字节类型的小端字节交换（AB CD → CD AB），常见于 AB PLC。
- DCBA：4 字节类型的小端（AB CD → DC BA）。

## decimal 缩放

`decimal` 字段对读到的数值乘以该系数后再上送。典型场景：

- 设备读数 250 表示 25.0 ℃ → decimal 设为 0.1
- 设备读数 1234 表示 12.34 MPa → decimal 设为 0.01

写值时同理，界面输入 25.0 会转换为 250 写入。

## bit 位（bool）

Modbus 线圈是 16-bit 一组，但实际通常按 bit 使用。`bit` 字段指定 0~15 中的一位。

例如：
- 地址 `00001`，bit=0 → 第 1 个线圈
- 地址 `00001`，bit=3 → 第 4 个线圈

## 性能

- **批量读取**：按 area 分组，组内连续寄存器自动合并为单次请求。
- **最大单次寄存器数**：120（Modbus 规范上限 125，留 5 个 buffer）。
- **gap 容差**：相邻区间 offset 差 < 8 寄存器会自动合并。
- **空闲超时**：默认 60s 空闲自动断开连接，下次读取时重连（可配）。

## 错误码

| 异常码 | 含义 |
|---|---|
| 01 | 非法功能码 |
| 02 | 非法数据地址 |
| 03 | 非法数据值 |
| 04 | 从设备故障 |
| 05 | 确认（仅用于 FC05/06）|
| 06 | 从设备忙 |
| 08 | 存储奇偶校验差错 |
| 0A | 网关路径不可用 |
| 0B | 网关目标无响应 |
